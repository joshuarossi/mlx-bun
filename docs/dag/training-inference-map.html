Scroll to zoom (Google-Maps style), drag to pan — out for structure, in for each node's oracle + allocate/free. Solid edges = the always-taken path. Dashed teal edges = a flag bridge: a CLI/env flag that routes you onto a different island (the label is the flag). Parity tier = the lowest layer whose oracle a node matches bit-for-bit: L1 = drop-in for mlx-lm, L2 = drop-in for mlx-optiq, L3 = our originals (no oracle → gated by finite-diff / KL / quality). Parity gates performance: a faster route only sits in a lower tier if its oracle exists AND it matches bit-exactly; otherwise it bubbles up. Trace a path to see where a flag drops you off the bit-exact floor into L3. Scope: MiniCPM5-1B. Per-model specialization means other models route differently — gemma-4 / e4b can take the compiled-decode path (gemma4-only here) and e4b adds a MoE expert-routing handler; a separate map per model is future work.
